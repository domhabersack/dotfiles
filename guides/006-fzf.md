# fzf: fuzzy finder everywhere

## What changed

fzf is now auto-installed into `~/.fzf` on first shell start if it isn't already present. It integrates into both zsh and vim/neovim.

### In zsh

| Binding | Action |
|---|---|
| `Ctrl-R` | Fuzzy search shell history — replaces the default reverse search |
| `Ctrl-T` | Insert a file path from the current directory tree |
| `Alt-C` | `cd` into a directory picked from the tree |

### In vim/neovim

CtrlP has been replaced by fzf. The same `Ctrl-P` muscle memory still works.

| Binding | Action |
|---|---|
| `Ctrl-P` | Open file picker (fzf over project files) |
| `<leader>b` | Fuzzy-search open buffers |
| `<leader>g` | Search file contents with ripgrep (`Rg`) |

Heavy directories (`.git`, `node_modules`, `.next`, `dist`) are excluded automatically. If `fd` is installed, fzf uses it; otherwise it falls back to `find`.

## Why it helps

`Ctrl-R` history search becomes genuinely useful once the results are fuzzy and sortable. You can type fragments of a command — even out of order — and narrow it down fast. The same applies to file picking: you don't need to remember the exact path or filename, just enough of it to filter.

`<leader>g` (ripgrep from within vim) lets you search across the entire project without leaving the editor and jump straight to the match.

## When to use it

- `Ctrl-R`: any time you want to re-run a command you've run before
- `Ctrl-P` in vim: any time you're opening a file by name
- `<leader>g`: searching for a symbol, string, or pattern across the codebase

## What to look out for

- fzf auto-installs on the first shell start on a new machine. If you're offline, the clone fails and the bindings won't be available until you install it manually (`~/.fzf/install`).
- `<leader>g` requires `rg` (ripgrep) to be installed. Without it, `:Rg` throws an error.
- `fd` is optional but makes fzf faster and respects `.gitignore` automatically. Install with `brew install fd`.
- On Terminus (iOS SSH), `Alt-C` may not fire depending on how the app maps the Option key.
